# 0437_path-sum-iii

遍历时需要维护的数据:

1. 路径上各个点的值. 是一个数组.
2. root到当前点的和.
3. 用map记忆: 路径上每一个点到root的和 -> 这个点于路径(1中维护的数组)的位置. 有些类似[0001_two-sum](https://leetcode.com/problems/two-sum/)那样用map记录的做法.

这样就能做到时间复杂度`O(n)`.

因为题目只要求路径个数而不是各条路径的详细信息, 所以上面的路径数组也可以略掉, 剩下的map的k-v的v则是一个代表有多少种**可能性**的值, 毕竟可能会有部分路径和为`0`.
